Notes On Implementation

 

It's possible that the current class library (TAL.Developer.Test.Domain) is being used by more than one application including this website. We also want to make our Domain knowledge available to all our applications on all our platforms so we:

 

1.      Created Domain class library with new RESTful HTTP Services Web API (TAL.Developer.Test.Domain)

a.      With core references to MVC and Web API

b.      Scaffold for unit tests for the new Web API (TAL.Developer.Test.Domain.Tests)

c.      Included NInject.3.3.4 in [Domain] and [Domain.Test] -> Help with IoC and SOLID priciples in general

d.      Included AutoMapper.8.1.1 in [Domain] and [Domain.Test] -> Help with Object mapping

e.      Installed Node.js (v10.16.2 LTS local x64) for Angular Site

f.       Included Microsoft.AspNet.WebApi.Cors 5.2.7 in API (Domain) project to handle CORS

2.      Created datastore (TAL.mdf) SQL Server 2016 and reference it in new Database project (TAL.Developer.Test.Database). Although it shouldn’t be needed as the database is included in the database a complete rebuild script can also be found in this project Create and Populate Script\ DBBuild.sql. Don’t forget to change the FILENAME! property in the script.

3.      Used Web Api Calls instead of referencing using the API directly to showcase flexibility through separation of concerns.

4.      Initially Angular for UI project was at version 5.2.5 based on Visual Studio 2017 project template - couldn't update (was targeting v8.3.2)

5.      npm install ng-pick-datetime@5.2.6 -save

6.      npm install @angular/animations --save
