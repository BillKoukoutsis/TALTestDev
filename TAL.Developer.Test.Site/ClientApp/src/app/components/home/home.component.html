TAL Developer Test Notes

 

Notes On Use

 

1.      Roles based security model.

2.      Admin  users have access to all menu items.

3.      All other user Roles have access to their respective entries in the List Timesheets menu item and the Timesheets Report menu item. List Groups, List Timezones and List Employees menu items are hidden to all other user Roles.

4.      List Groups menu item is visible to users in Admins group only. Allows for the creation and edit of new and existing groups. The Admins and Users group however cannot be modified.

5.      List Timezones menu item is visible to users in Admins group only. Allows for the creation and edit of new and existing timezones. The 'E. Australia Standard Time' timezone however cannot be modified.

6.      List Employees menu item is visible to users in Admins group only. Allows for the creation and edit of new and existing employees. The 'Admin User' employee however cannot be modified.

7.      List Timesheets menu item is visible by all Roles. Users in the Admins group see a list of all timesheets for all users across all time. Users in all other groups see a list of timesheets for their profile only across all time.

8.      Timesheets report menu item is visible by all Roles. Users in the Admins group see a list of timesheets data for all users for the specified date. Users in all other groups see timesheet data for their profile only and for the selected date.

 

The same application is used to emulate transactional input (clock in, clock out, etc), posting transactional data via a simple rest based API.

Notes on Tables

 

1.      tblGroups table

a.      grpId int pri key

b.      grpName varchar(50)

c.      2 groups that can't be deleted / edited or added just viewed

                                                    i.     Group 1 is Admins

                                                   ii.     Group 2 is Users

2.      tblTimezones table

a.      tmzId int pri key

b.      tmzName varchar(150) -> Will currently be one timezone 'E. Australia Standard Time'

NOTE: Valid timezone names can be found at https://dzone.com/articles/dates-and-times-in-sql-server-at-time-zone

 

3.      tblEmployees table

a.      empId int pri key

b.      empName varchar(150)

c.      empUsername varchar(50)

d.      empPassword varchar(50) <-stored encrypted

e.      empGroupId int foreign key <- ((2)) 'Users' by default

f.       empTimezoneId int foreign key <- ((1)) ''E. Australia Standard Time'' by default

g.      empRate decimal(12,2) <- 0.00 by default

h.      1 employee initially pre-created

                                                    i.     empId=1

                                                   ii.     empName='Admin User'

                                                  iii.     empUsername ='admin'

                                                  iv.     empPassword='admin'

                                                   v.     empGroupId = 1 (Admins)

                                                  vi.     empTimezoneId = 1 ('E. Australia Standard Time')

                                                vii.     empRate = 0.00

i.       Employees can be created and edited. They can't be deleted.

j.       Employees can only be created by Admin's. (user and passwords initially set by admins).

k.      Employees can change details (username/password) after they log in (will require logout / login).

4.      tblTimesheets table

a.      tshId int pri key

b.      tshEmployeeId int foreign key

c.      tshTimezoneId int foreign key -> at moment only ever 'E. Australia Standard Time' -> based on employees timezone at StartDate -> if left null then assume employees default timezone

d.      tshStartDate -> Utc Date Time -> there can only be one per day per employee

e.      tshEndDate -> Utc Date Time -> Must be greater than tshStartDate but same day as tshStartDate

f.       tshRate -> based on employees rate at StartDate -> if left null then assume employees default rate. Overrides employees rate if different.

  

Notes on Screens

 

Regarding 'There should be a small application to emulate transactional input (clock in, clock out, etc), posting transactional data via a simple rest based API'. This has been integrated with the main site for convenience (see Timesheet edit screen below).

 

1.      Groups List Screen

a.      Only Admins group can list Groups

b.      Users group cannot list Groups

 

2.      Group Edit Screen

a.      Only Admins group can edit a Group EXCEPT the Admins and Users group

b.      Users group cannot edit a Group

c.      No one can delete a Group

 

3.      Timezones List Screen

a.      Only Admins group can list Timezones

b.      Users group cannot list Timezones

 

4.      Timezone Edit Screen

a.      Only Admins group can edit a Timezone EXCEPT the 'E. Australia Standard Time' Timezone

b.      Users group cannot edit a Timezone

c.      No one can delete a Timezone

 

5.      Employees List Screen

a.      Only Admins group can list ALL Employees

b.      Users group cannot list Employees EXCEPT their own account

 

6.      Employee Edit Screen

a.      Only Admins group can edit an Employee EXCEPT the Admin user

b.      Users group cannot edit a User EXCEPT for their own account

c.      No one can delete a Group

 

7.      Timesheets List Screen

a.      Only Admins group can list ALL Timesheets

b.      Users group cannot list Timesheets EXCEPT for their own account

 

8.      Timesheet Edit Screen

a.      Only Admins group can edit a Timesheet EXCEPT the Admin user

b.      Users group cannot edit a Timesheet EXCEPT for their own account AND for the current day

c.      No one can delete a Timesheet

 

9.      Timesheets Report Screen (By Day)

a.      Only Admins group can report on ALL Timesheets

b.      Reporting uses the 'E. Australia Standard Time' Timezone as the reference timezone when converting Utc to Local time.
